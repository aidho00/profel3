04 Big Idea

Tables Should Not Live Alone

In real systems, information is connected.

A student belongs to a course. A grade belongs to a student. A payment belongs to an account. These connections are represented using keys.

06 Simple Definition

What Is a Foreign Key?

A foreign key is a column that points to the primary key of another table.

Simple version: it is a connector. It tells the database that a value in one table must match a valid record in another table.

08 Parent Table

Courses Table

The courses table is the parent table because its course_id will be referenced by students.

CREATE TABLE courses (
  course_id INT PRIMARY KEY AUTO_INCREMENT,
  course_code VARCHAR(20) NOT NULL UNIQUE,
  course_name VARCHAR(100) NOT NULL
);
09 Child Table

Students Table

The students table is the child table because each student must choose a valid course from the courses table.

CREATE TABLE students (
  student_id INT PRIMARY KEY AUTO_INCREMENT,
  student_number VARCHAR(20) NOT NULL UNIQUE,
  last_name VARCHAR(50) NOT NULL,
  first_name VARCHAR(50) NOT NULL,
  course_id INT NOT NULL
);

12 Why Not Store Course Name Directly?

The Repetition Problem

Without relationship

BSIS, B.S.I.S., BS Info Systems, Information System

With relationship

Store only course_id and get the official course details from courses.

A foreign key helps keep values consistent and easier to update.

13 Vocabulary

Foreign Key Words to Remember

Primary key: unique ID in parent table
Foreign key: reference in child table
Parent table: table being referenced
Child table: table containing the reference
14 Rule

The Foreign Key Must Match

A student cannot reference a course that does not exist.

If courses only have IDs 1, 2, and 3, the students table should not accept course_id 99.

15 Referential Integrity

What Is Referential Integrity?

Referential integrity means relationships between tables stay valid and consistent.

It protects the database from broken links, like a student record pointing to a non-existing course.

17 SQL Syntax

Adding a Foreign Key Constraint

A constraint is a database rule. This rule says: students.course_id must match courses.course_id.

CONSTRAINT fk_student_course
  FOREIGN KEY (course_id)
  REFERENCES courses(course_id)
18 Complete Example

Students Table with Foreign Key

CREATE TABLE students (
  student_id INT PRIMARY KEY AUTO_INCREMENT,
  student_number VARCHAR(20) NOT NULL UNIQUE,
  last_name VARCHAR(50) NOT NULL,
  first_name VARCHAR(50) NOT NULL,
  course_id INT NOT NULL,

  CONSTRAINT fk_student_course
    FOREIGN KEY (course_id)
    REFERENCES courses(course_id)
);

23 Common Error Message

Cannot Add or Update Child Row

Cannot add or update a child row: a foreign key constraint fails

This error usually means the foreign key value you inserted does not exist in the parent table.

24 Delete Protection

Can We Delete a Used Course?

If students are already assigned to BSIS, should we delete the BSIS course record?

Referential integrity may block deletion because deleting the parent record would break existing student records.

25 Common Error Message

Cannot Delete Parent Row

Cannot delete or update a parent row: a foreign key constraint fails

This means another table is still using that record. Check the child table first.

29 Join Preview

How to Display the Course Name

Because students only store course_id, we use a join to display the course code and course name.

SELECT
  students.student_number,
  students.first_name,
  courses.course_code
FROM students
INNER JOIN courses
  ON students.course_id = courses.course_id;

34 Common Mistake 1

Different Data Types

Wrong

courses.course_id is INT but students.course_id is VARCHAR.

Better

Both course_id columns should use compatible data types.

Foreign key columns should match the referenced primary key type.

35 Common Mistake 2

Creating Tables in Wrong Order

Cannot add foreign key constraint

This can happen when the parent table does not exist yet, the referenced column is not indexed, or the data types do not match.

36 Common Mistake 3

Inserting Child Before Parent

You must add the course before assigning a student to that course.

The database checks the parent table first. If the referenced course is not there, the student record is rejected.

40 Lesson Summary

What We Learned

  • A foreign key connects one table to another table.
  • The parent table contains the referenced primary key.
  • The child table contains the foreign key.
  • Referential integrity prevents invalid links.
  • Foreign keys are useful in real applications like course dropdowns.
