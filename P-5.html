02 Warm-up

Quick Review

What is the difference between a database and a DBMS?

A database stores the organized data. A DBMS, such as MySQL or MariaDB, manages the database and allows users to create, search, update, and control records.

database = stored data
DBMS = manager software

06 Rows

A Row Is One Complete Record

A row should describe one real item, person, event, or transaction. In a students table, one row should describe one student.

Example row: 2026-001 · Cruz · Ana · BSIS · 1

Do not split one student into many rows unless the table has a different purpose, such as enrollment records or grade records.

07 Columns

A Column Stores One Kind of Detail

A column should contain the same type of information for every row.

student_number
last_name
first_name
course_id
year_level
email

Good column design makes searching, sorting, and validation easier.

08 Cells

A Cell Is One Value

A cell is the intersection of one row and one column. It should usually contain only one value.

Good cell

first_name: Ana

last_name: Cruz

Problem cell

name: Ana Cruz, 0917-111-2222, BSIS

This mixes different details in one cell.

09 Table Design Rule

One Column, One Meaning

Do not use one column for different kinds of data. A database becomes easier to search when each column has one clear meaning.

❌ contact
May contain phone, email, Facebook name, or address.
✅ email and contact_number
Each column has a clear purpose.
10 Naming Columns

Use Clear and Consistent Names

Column names should be understandable. Avoid spaces and avoid names that are too vague.

Better names

  • student_number
  • first_name
  • last_name
  • birth_date

Avoid

  • student no.
  • fname?
  • data1
  • date
11 Naming Style

Recommended Naming Convention

For this course, we will use lowercase names with underscores. This is readable and beginner-friendly.

-- recommended table and column names
students
student_id
student_number
last_name
first_name
course_id

Consistency matters more than style. Once a naming style is chosen, use it throughout the database.

12 Data Types

What Is a Data Type?

A data type tells the database what kind of value a column can store.

For example, age is a number, name is text, birth date is a date, and amount due is a decimal value.

number
text
date
decimal
true/false
13 Why Types Matter

Data Types Protect the Table

If a column has the correct data type, the database can store values properly and prevent many mistakes.

  • Numbers can be calculated.
  • Dates can be sorted by time.
  • Text can be searched by name or word.
  • Decimal values can store money more correctly.
  • Wrong values become easier to detect.
14 Common MySQL Types

The First Types to Know

INT

Whole numbers like ID, age, quantity, or year level.

VARCHAR

Variable-length text such as names, emails, and addresses.

CHAR

Fixed-length text like short codes with consistent length.

DATE

Calendar dates such as birth dates and enrollment dates.

DECIMAL

Numbers with exact decimal places, often used for money.

BOOLEAN

True/false values, often stored as 1 or 0.

15 INT

Use INT for Whole Numbers

INT stores whole numbers. It is commonly used for IDs, counts, quantities, and year levels.

student_id INT
year_level INT
quantity INT
Do not use INT for phone numbers because phone numbers may start with 0 and are not used for calculations.
16 VARCHAR

Use VARCHAR for Flexible Text

VARCHAR stores text with variable length. The number inside parentheses is the maximum number of characters.

last_name VARCHAR(50)
email VARCHAR(100)
address VARCHAR(255)

Use VARCHAR for names, addresses, descriptions, and other text that can vary in length.

17 CHAR

Use CHAR for Fixed-Length Text

CHAR is useful when the value always has the same length or a very predictable format.

Possible CHAR use

sex CHAR(1)

M or F

Possible VARCHAR use

course_code VARCHAR(20)

BSIS, BSAIS, BSCRIM

For beginners, VARCHAR is safer for most text fields unless the teacher requires CHAR.

18 DATE

Use DATE for Calendar Dates

DATE stores dates in a format that the database can sort and compare.

birth_date DATE
enrollment_date DATE

-- MySQL date format
'2026-07-18'
The usual MySQL date format is YYYY-MM-DD.
19 DECIMAL

Use DECIMAL for Money and Exact Values

DECIMAL stores exact numbers with decimal places. It is better for money than approximate floating-point values.

tuition_fee DECIMAL(10, 2)
grade DECIMAL(4, 2)

DECIMAL(10,2) means up to 10 digits total, with 2 digits after the decimal point.

20 BOOLEAN

Use BOOLEAN for Yes or No Values

BOOLEAN represents true or false. In MySQL/MariaDB, it is often stored internally as 1 or 0.

is_active BOOLEAN
is_graduated BOOLEAN
has_scholarship BOOLEAN
1 = true
0 = false

22 Important Choice

Phone Numbers Are Usually Text

A phone number may look numeric, but we usually store it as text.

❌ INT
09171234567 may lose the first zero or become too large.
✅ VARCHAR(20)
Stores 0917-123-4567 exactly as typed.

Rule: If you do not calculate it, it might be better as text.

23 Important Choice

Student Numbers Are Usually Text

Student numbers may contain dashes, letters, or leading zeros. Because of this, they are usually stored as VARCHAR.

student_number VARCHAR(20)

-- examples
'2026-001'
'BSIS-24-0007'
'00012345'

26 Relationship Preview

Why course_id Instead of course_name?

Later, we will create a separate courses table. The students table will store course_id to connect each student to a course.

coursescourse_id, course_code, course_name
1 → many
studentsstudent_id, name, course_id

This avoids repeatedly typing long course names for every student.

27 Sample SQL

Students Table Draft

Do not worry if this looks new. We will formally study CREATE TABLE later. Today, use this as a preview of how table design becomes SQL.

CREATE TABLE students (
  student_id INT PRIMARY KEY AUTO_INCREMENT,
  student_number VARCHAR(20),
  last_name VARCHAR(50),
  first_name VARCHAR(50),
  course_id INT,
  year_level INT
);

29 Common Mistake

Too Many Details in One Column

Problem design

student_info

2026-001 Ana Cruz BSIS 0917...

Better design

student_number

first_name

last_name

course_id

Separate details into separate columns so the database can search and update them correctly.

30 Common Mistake

Vague Column Names

Avoid column names that are unclear. A future programmer or classmate should understand the table without guessing.

❌ data1, data2, info, value, text
✅ student_number, last_name, first_name, contact_number
Clear names make database maintenance easier.
