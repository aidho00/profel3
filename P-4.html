03 Big Idea

A DBMS Manages the Database

A database stores organized data. A DBMS is the software that helps users create, manage, update, search, and protect that data.

Database

The organized collection of related data.

DBMS

The software that controls and manages the database.

04 Analogy

Database vs DBMS

🗄️

Database

Like a filing cabinet that stores records.

👨‍💼

DBMS

Like the office staff who organize, retrieve, update, and control access to the records.

The cabinet stores. The staff manages. In the same way, the database stores data, while the DBMS manages it.

05 Definition

What Is a DBMS?

A Database Management System is software used to create, store, organize, retrieve, update, and secure data in a database.

Examples include MySQL, MariaDB, PostgreSQL, Oracle Database, Microsoft SQL Server, SQLite, and MongoDB.

07 What a DBMS Does

Main Responsibilities

  • Creates databases and tables.
  • Stores records in a structured format.
  • Retrieves information using SQL queries.
  • Checks rules such as primary keys and required fields.
  • Controls user access and helps protect the data.

10 Database Types

Why Are There Different Types?

Different systems have different needs. Some systems need strict tables, some need fast document storage, and others need flexible data formats.

banking records
social media posts
inventory data
medical records

Because data can be organized in different ways, several DBMS models were created.

11 Type 1

Hierarchical DBMS

🌳

Tree Structure

Data is arranged like a parent-child tree.

Simple Idea

One parent can have many children, but each child usually belongs to one parent.

Example analogy: a school has departments, departments have programs, and programs have students.

12 Hierarchical Model

Strengths and Limits

Strength

Easy to understand when the data naturally follows a tree structure.

Limit

It becomes difficult when one record needs to connect to many different parents.

This model is important historically, but it is not the main focus of this course.

13 Type 2

Network DBMS

🕸️

Web-Like Structure

Records can have multiple connections to other records.

Simple Idea

A child record may connect to more than one parent record.

This is more flexible than the hierarchical model, but it can be harder for beginners to visualize and manage.

14 Network Model

Where It Helps

A network model can represent complex connections. However, the structure can become hard to maintain when the system grows.

many connections
complex paths
harder design
less beginner-friendly
15 Type 3

Relational DBMS

A relational DBMS organizes data into tables that can be connected using keys.

This is the most important model for our course because MySQL/MariaDB uses the relational approach.

tables
rows
columns
relationships
16 Relational Model

Why It Is Beginner-Friendly

  • Data is arranged in tables similar to spreadsheets.
  • Each table has a clear purpose.
  • Rows represent records.
  • Columns represent fields.
  • Tables can connect through primary and foreign keys.
17 Type 4

Object-Oriented DBMS

📦

Object Storage

Data is stored as objects, similar to how programming languages use objects.

Use Case

Can be useful when applications work heavily with complex objects and classes.

We will not focus on this model, but it helps students see that database design can match programming styles.

18 Type 5

NoSQL DBMS

🧾

Flexible Data

NoSQL systems may store data as documents, key-value pairs, wide columns, or graphs.

Use Case

Often used for flexible, large-scale, or rapidly changing data.

Examples include MongoDB, Redis, Cassandra, and Neo4j.

19 NoSQL Reminder

Not Bad, Just Different

NoSQL does not mean SQL is useless. It means the database may not follow the traditional relational table model.

For this course, relational databases are better because they teach tables, keys, relationships, and SQL clearly.

21 Course Focus

Why Relational Databases?

  • They are widely used in school, business, banking, and government systems.
  • They organize data clearly using tables.
  • They support strong rules using keys and constraints.
  • They use SQL, which is a very important database language.
  • They connect well with Java applications using JDBC.
22 Relational Vocabulary

Relation Means Table

In database theory, the word relation often means a table. For beginners, it is okay to think of a relation as a table of related records.

Relation

A table that stores one kind of record.

Relational Database

A database made of related tables.

24 Records

What Is a Row?

A row is one complete record inside a table.

In a students table, one row describes one student. In a courses table, one row describes one course.

What Is a Column?

A column is one field or category of information in a table.

Examples of student columns are student_number, last_name, first_name, course_id, and year_level.

26 Schema

What Is a Database Schema?

A schema is the planned structure of the database. It describes the tables, columns, data types, keys, and relationships.

Blueprint

The schema is like the blueprint of a building.

Actual Data

The records are like the people and objects inside the building.

28 Relationships

Why Tables Connect

In a relational database, we avoid repeating long information over and over. Instead, we store related information in separate tables and connect them.

Example: Store course details once in the courses table, then connect students to their course using course_id.

30 Keys Preview

Primary Key and Foreign Key

Primary Key

A unique identifier for a record, such as course_id.

Foreign Key

A column that points to a record in another table.

We will study keys more deeply in a later lesson, but today you only need to see how tables connect.

31 SQL Preview

Joining Related Tables

A join combines related records from two tables.

SELECT students.first_name, courses.course_code
FROM students
JOIN courses
  ON students.course_id = courses.course_id;

Repeating long course names makes updates harder. A separate courses table is cleaner.

33 Cleaner Design

Use a Course Table

courses table

Stores course_id, course_code, and course_name once.

students table

Stores student details and course_id as the link.

This is one reason relational databases are powerful: each table can focus on one type of record.

38 Change One Thing

Improve the Design

Suppose the students table stores the full course name as text. What can we change?

Create a separate courses table, then store course_id in the students table.

This small design change introduces relationships and reduces repeated data.

39 Common Misunderstanding

phpMyAdmin Is Not the DBMS

phpMyAdmin

A visual tool in the browser for managing databases.

MySQL/MariaDB

The DBMS or database service that stores and processes the data.

Closing the browser does not mean the database disappears. The data is handled by the database service.
