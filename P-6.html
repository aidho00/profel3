05 Key Vocabulary

What Is a Key?

A key is a column, or group of columns, used to identify records in a table.

The key tells the database: “This is the row I mean.”

unique
not confusing
used for searching
used for relationships
06 Main Types

Three Keys for Today

Primary KeyThe chosen main identifier of the table.
Candidate KeyA possible identifier that could uniquely identify a row.
Composite KeyA key made from two or more columns.
Unique IdentifierA value that separates one record from all others.
Primary Key

The primary key is the table’s main ID.

Each table usually has one primary key. It must identify only one row.

07 Primary Key Rules

A Primary Key Must Be...

UniqueNo two rows should have the same value.
Not NullIt cannot be blank or missing.
StableIt should not keep changing.
SpecificIt should identify one exact record.

09 SQL Pattern

Declaring a Primary Key

CREATE TABLE students (
  student_id INT PRIMARY KEY AUTO_INCREMENT,
  student_number VARCHAR(20),
  last_name VARCHAR(50),
  first_name VARCHAR(50)
);

AUTO_INCREMENT lets MySQL automatically create the next number.

10 Tiny Explanation

What Does AUTO_INCREMENT Do?

First insertstudent_id becomes 1
→
Next insertstudent_id becomes 2, then 3, then 4...

Students do not need to manually type the internal ID.

Candidate Key

A candidate key is a possible primary key.

A table may have more than one column that can uniquely identify records.

13 Choosing the Primary Key

One Candidate Becomes Primary

Candidate Keysstudent_id, student_number, email
→
Chosen Primary Keystudent_id

The other candidate keys can still be protected using UNIQUE constraints.

14 UNIQUE Constraint

Protect Other Candidate Keys

CREATE TABLE students (
  student_id INT PRIMARY KEY AUTO_INCREMENT,
  student_number VARCHAR(20) UNIQUE,
  email VARCHAR(100) UNIQUE
);

UNIQUE means the database should reject duplicate values in that column.

15 Quick Check

Ask: Can two students have the same value? If yes, it is not safe as a candidate key.

Composite Key

A composite key uses two or more columns together.

Sometimes one column alone is not enough, but a combination is unique.

16 Composite Example

Enrollment Records

student_idsubject_idterm_idstatus
11012026-1Enrolled
11022026-1Enrolled
21012026-1Enrolled

student_id alone repeats. subject_id alone repeats. Together with term_id, the record becomes specific.

17 Composite SQL

Declaring a Composite Key

CREATE TABLE enrollments (
  student_id INT,
  subject_id INT,
  term_id INT,
  status VARCHAR(20),
  PRIMARY KEY (student_id, subject_id, term_id)
);

The combined values identify one enrollment row.

18 Composite Key Logic

Why Combine Columns?

student_idRepeats because one student can enroll in many subjects.
subject_idRepeats because many students can take the same subject.
term_idRepeats because many enrollments happen in the same term.
CombinedTogether, they describe one specific enrollment.
19 Natural Key

Natural Key: Already Exists in Real Life

A natural key is a real-world value used to identify a record.
student_number
employee_number
ISBN
license_number

Natural keys are meaningful, but they may change if the organization changes its numbering system.

20 Surrogate Key

Surrogate Key: Created by the Database

A surrogate key is an artificial ID created only for the database.
student_id INT PRIMARY KEY AUTO_INCREMENT

It has no meaning outside the database, but it is simple, short, and stable.

22 Bad Key Choices

Avoid These as Primary Keys

NameCan repeat and may change.
AddressLong, inconsistent, and changeable.
Phone numberCan change or be shared.
Course nameMany students can have the same course.
23 Good Key Qualities

How to Choose a Good Key

UniqueEvery row has a different value.
ShortEasy for the database to store and compare.
StableDoes not change often.
RequiredAlways has a value, never blank.

26 Courses Table

Primary Key in courses

CREATE TABLE courses (
  course_id INT PRIMARY KEY AUTO_INCREMENT,
  course_code VARCHAR(20) UNIQUE,
  course_name VARCHAR(100) NOT NULL
);

course_id is the main ID. course_code is also protected from duplicates.

27 Students Table

Primary Key in students

CREATE TABLE students (
  student_id INT PRIMARY KEY AUTO_INCREMENT,
  student_number VARCHAR(20) UNIQUE,
  last_name VARCHAR(50) NOT NULL,
  first_name VARCHAR(50) NOT NULL
);

The primary key identifies the row even if two students have similar names.

28 What Happens with Duplicates?

Duplicate Primary Keys Are Rejected

-- This should not be allowed
student_id: 1
student_id: 1
The database protects the table by refusing duplicate primary key values.
29 What Happens with NULL?

Primary Keys Cannot Be Blank

-- Not allowed for a primary key
student_id: NULL

If the key is missing, the database cannot reliably identify the record.

Do Not Use Changing Values

Phone numberStudents can change SIM cards.
AddressStudents can move to another place.

34 Common Mistake

Do Not Make Every Column Unique

Only values that must never repeat should be UNIQUE.

ColumnShould it be UNIQUE?Why?
student_numberYesOne official number per student
last_nameNoMany students may share a last name
course_idNo in studentsMany students can belong to the same course

36 Recap

What We Learned Today

  • A key helps identify one exact row.
  • A primary key is the chosen main identifier.
  • A candidate key is a possible identifier.
  • A composite key uses multiple columns together.
  • Good keys are unique, stable, required, and not confusing.
