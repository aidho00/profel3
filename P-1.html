W = Windows (Operating System)

A = Apache (Web Server)

M = MySQL/MariaDB (Database)

P = PHP (Scripting Language)

It bundles everything you need to run a local database server on your Windows machine.

Fundamentals of Database Systems 02 Foundations

What is Data?

Data is raw, unprocessed facts. Numbers, text, dates, images — anything that can be recorded. By itself, data has no meaning.

Examples of Data

"28" · "Juan" · "2024-09-01" · "Cebu"

Not Yet Useful

Without context, these are just isolated values.

Fundamentals of Database Systems 02 Foundations

What is Information?

Information is data that has been processed, organized, and given context. It answers questions and supports decisions.

"Juan dela Cruz, age 28, enrolled on September 1, 2024 at the Cebu campus."

Same data, now meaningful because of context and structure.

Fundamentals of Database Systems 02 Core Concept

What is a Database?

A database is an organized collection of related data that can be easily accessed, managed, and updated.

Think of it as a super-powered filing cabinet that can instantly find any record you need.

Fundamentals of Database Systems 02 Analogy

Database = Digital Filing Cabinet

📁 Physical Filing

Folders in a cabinet, organized by category. Finding one record means flipping through papers.

💾 Database

Tables in a system, organized by relationships. Finding one record takes milliseconds.

Fundamentals of Database Systems 02 Example: Banking

When You Use an ATM:

  • Step 1: Card data verified against Accounts table
  • Step 2: PIN checked against Security table
  • Step 3: Balance retrieved from Balances table
  • Step 4: Withdrawal recorded in Transactions table
  • Step 5: Balance updated instantly

All of this happens in under 2 seconds. That's database power.

Fundamentals of Database Systems 02 Key Concept

What is a DBMS?

Database Management System — the software that lets you create, read, update, and delete data in a database.

The DBMS sits between you and the raw data. You talk to the DBMS, and it handles the storage.

Fundamentals of Database Systems 02 Benefits

Why Use a DBMS?

  • Data Independence: Change storage without changing apps
  • Data Integrity: Rules prevent bad data from entering
  • Security: Control who sees what
  • Concurrent Access: Multiple users, no conflicts
  • Backup & Recovery: Never lose critical data

Fundamentals of Database Systems 02 History

A Brief History of Databases

  • 1960s: Flat files & hierarchical databases (IBM IMS)
  • 1970: E.F. Codd proposes the Relational Model
  • 1970s–80s: SQL invented, Oracle & IBM DB2 launched
  • 1995: MySQL released (open source revolution)
  • 2000s–Now: NoSQL, Cloud DBs, NewSQL

Fundamentals of Database Systems 02 Our Focus

The Relational Model

Proposed by Edgar F. Codd in 1970. Data organized in tables (relations) with rows (tuples) and columns (attributes).
